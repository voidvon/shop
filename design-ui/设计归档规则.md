# Design UI 归档规则

## 文件分工
- `app-shell.pen`：维护状态栏、底部导航、通用头部、按钮、卡片等壳层组件
- `shop-main.pen`：维护首页、分类、搜索、商品列表、商品详情等商城主链路页面
- `order-flow.pen`：维护购物车、确认订单、支付结果、订单列表、订单详情等订单流页面
- `account.pen`：维护我的、登录注册、设置、地址、优惠券等账户域页面

## 新页面归档
- 进入商城主流程的页面，优先放入 `shop-main.pen`
- 与下单、支付、订单状态强相关的页面，放入 `order-flow.pen`
- 与用户信息、资产、账户设置强相关的页面，放入 `account.pen`

## 组件维护
- 当前 Pencil 环境下，跨 `.pen` 文件 `imports + ref` 不稳定，不作为正式维护链路
- `app-shell.pen` 作为公共组件源文件
- 如果某业务文件需要复用公共组件，先从 `app-shell.pen` 复制组件母版到目标文件，再在目标文件内用 `ref` 实例化
- 同一业务文件内，尽量保证同类组件只有一份母版定义

## 页面命名
- 使用 `中文业务名 / EnglishKey` 形式，例如：`商品详情 / Product Detail`
- 页面状态稿统一追加状态后缀，例如：`分类 / Category / Empty`、`购物车 / Cart / Loading`

## 维护原则
- 每个业务文件尽量控制在 5-8 张主要页面以内
- 公共组件先抽壳层，再考虑抽业务组件
- 删除旧稿前，先确认新文件已通过截图和布局检查
